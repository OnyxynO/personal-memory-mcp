personal-memory-mcp — Architecture technique

Document de référence pour comprendre les deux modes d'alimentation du serveur MCP.

1. Vue d'ensemble — Ce qu'est personal-memory-mcp

Un serveur MCP local qui centralise une mémoire personnelle et l'expose à n'importe quel client IA compatible MCP (Claude Code, Claude Desktop, Cursor, etc.).

Principe : Ollama est utilisé uniquement pour les embeddings vectoriels (calcul de similarité). L'extraction des faits peut venir de deux sources différentes — c'est l'objet de ce document.

2. Mode A — Import batch via qwen3 (existant)

L'utilisateur lance mmcp import manuellement. qwen3:1.7b lit les conversations passées et extrait les faits. L'IA cliente n'est pas impliquée.

(Ollama) participant EMB as nomic-embed-text
(Ollama) participant DB as SQLite + sqlite-vec User->>CLI: mmcp import claude-code CLI->>FS: Scan ~/.claude/projects/**/*.jsonl FS-->>CLI: 207 fichiers JSONL loop Pour chaque session CLI->>Q: Texte conversation (tronqué 2000 chars/msg) Q-->>CLI: JSON [{contenu, categorie, score_confiance}] CLI->>EMB: Batch encode(faits[]) EMB-->>CLI: embeddings[] CLI->>DB: Déduplique + insère faits end CLI-->>User: +35 faits, =1 dédupliqué, 58s

3. Mode B — Import batch via l'IA cliente (proposé)

L'IA déjà connectée au serveur MCP (Claude, GPT, peu importe) lit elle-même les conversations brutes et appelle add() pour chaque fait qu'elle juge mémorisable. Zéro qwen3.

sequenceDiagram participant User as Utilisateur participant AI as IA cliente
(Claude, GPT, Cursor...) participant MCP as personal-memory-mcp participant FS as Fichiers
(JSONL / ZIP) participant EMB as nomic-embed-text
(Ollama) participant DB as SQLite + sqlite-vec User->>AI: "Importe mes conversations Claude Code" AI->>MCP: import_conversations(source="claude-code", page=1) MCP->>FS: Lit et découpe les conversations FS-->>MCP: Chunk 1/N (texte brut) MCP-->>AI: {conversations: [...], page: 1, total_pages: N} loop Pour chaque chunk reçu AI->>AI: Décide quoi mémoriser AI->>MCP: add("Préfère uv plutôt que pip", "preference") MCP->>EMB: encode(fait) EMB-->>MCP: embedding MCP->>DB: Déduplique + insère MCP-->>AI: {id: 42, nouveau: true} end AI->>MCP: import_conversations(source="claude-code", page=2) Note over AI,MCP: Continue jusqu'à la dernière page AI-->>User: "J'ai mémorisé 12 faits depuis tes sessions"
Avantage clé : L'IA qui fait l'import est la même que celle qui utilisera la mémoire. Elle sait exactement ce qui sera utile dans ses futures sessions.

Nouvel outil MCP nécessaire : import_conversations

4. Mode C — Alimentation temps réel (outil add existant)

Pendant une session normale, l'IA appelle add() dès qu'elle apprend quelque chose de durable. Pas d'import, pas de batch — la mémoire se construit au fil des conversations.

sequenceDiagram participant User as Utilisateur participant AI as IA cliente participant MCP as personal-memory-mcp participant EMB as nomic-embed-text participant DB as SQLite + sqlite-vec User->>AI: "Je préfère toujours travailler avec uv plutôt que pip" AI->>AI: Détecte une préférence durable AI->>MCP: add("Préfère uv plutôt que pip", "preference") MCP->>EMB: encode(fait) EMB-->>MCP: embedding MCP->>DB: Déduplique + insère MCP-->>AI: {id: 43, nouveau: true} AI-->>User: "Je retiens ça pour nos prochaines sessions." Note over User,AI: Session suivante... User->>AI: Nouvelle session démarre AI->>MCP: search("outils python préférés") MCP->>DB: Recherche vectorielle DB-->>MCP: [{contenu: "Préfère uv plutôt que pip", score: 0.94}] MCP-->>AI: faits pertinents AI->>AI: Injecte dans son contexte AI-->>User: (sait déjà que l'utilisateur préfère uv)
Prérequis : L'IA doit être instruite (via system prompt ou description des outils) de quand appeler add. Sans instruction explicite, elle ne le fait pas spontanément.

7. Ce qu'il faut implémenter pour le Mode B

  1. Nouvel outil MCP import_conversations
    Lit les JSONL ou ZIP, découpe en pages, retourne le texte brut à l'IA. Pas d'appel LLM côté serveur — juste du parsing et du découpage.
  2. Pagination obligatoire
    207 sessions × ~50 messages = trop long pour un seul appel. Pages de 3 à 10 conversations selon la taille.
  3. Description des outils MCP
    Les descriptions des outils add et import_conversations doivent expliquer clairement quand les appeler (injectées automatiquement par le protocole MCP).
Résumé : qwen3 reste utile comme fallback pour les utilisateurs sans IA connectée ou qui veulent un import 100% local. Le Mode B devient le mode recommandé quand une IA est disponible.
